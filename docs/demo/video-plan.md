# Demo 计划

## 目标

向评委展示 Roundtable 的完整真实链路——**一句话需求 → PM 结构化计划 → 人工批准 → 多 Agent 分阶段执行 → 真实产物预览 → 结构化交接记录**——并逐段对应比赛评分维度（AI 协作 30% / 功能完整 25% / 生成效果 20% / 创新产品感 10%）。核心论点：别的工具是"一个 Agent 干活"，Roundtable 是"一个 Agent 团队怎么协作不失控"。

## 页面 / App

- 本地地址：**http://localhost:3000**（`corepack pnpm ui:dev`）
- 录制前置（必须全过，详见 `docs/demo/storyboard.md` Pre-flight）：
  1. `docker ps` 确认 postgres/redis 在跑（否则 `pnpm dev:services`）
  2. `curl -m12 https://api.deepseek.com` 返回任意 HTTP 码即可（401 正常）
  3. 用 dev 凭据登录（登录态才有 workbench/pin/自建 Agent 全功能）
  4. 发一条丢弃用消息，确认计划卡**无 `degraded` 标记**
- 窗口 1280×800+，浅色主题，关闭系统通知，**全程不出现 `.env` / API key / Deploy 字样**

## 演示要求

1. 打开 http://localhost:3000，停在首页圆桌全景 2 秒
2. 侧栏点 **New task**，输入 prompt：`做一个产品定价页，含月付/年付切换`，回车发送
3. 等 PM 出计划卡（约 3–8 秒，**保留这个等待**，让评委看到"先计划不动手"）→ 点 **Show plan** 展开
4. 点 **Approve** → 等 StageCards 逐阶段出现（Build → Review，期间 20–40 秒，**剪辑加速到 5 秒内**，保留阶段卡翻面的瞬间）
5. 鼠标指向阶段卡右上角 **平台 chip**（"Claude Code"），停顿 1 秒
6. 展开 code 产物 → 点右上 **复制按钮**（变绿色对勾，放大特写）→ 切到 preview 产物，iframe 里直接交互月付/年付切换（**放大**）
7. **上下文一幕**：hover PM 回复点 **pin 按钮**（高亮特写）→ 在输入框发追问 `把切换做成能记住用户的选择`（一句没头没尾的话）→ 等新计划卡出现，**放大 intake 摘要**——它准确说出了"给定价页切换加状态持久化"
8. 顶部进 **Workflow 编辑器**：拖一个阶段、点开 Ship 阶段的 user-approval 闸口设置，停 3 秒即可，不深玩
9. 切到编辑器/终端打开 **`ai-logs/handoffs.jsonl`**，滚动到今天的真实记录，**放大 `context_audit` 字段** 2 秒
10. 回到首页全景收尾
- 兜底：若 LLM 当场异常，步骤 2–7 全部改用登出态脚本剧场录制，台词去掉"真实模型"一句，其余不变

## 逐字稿

1. 旁白：这是 Roundtable——把多个 Coding Agent 拉到一张圆桌旁，像一个真实团队一样协作。
   画面：首页圆桌全景，光标扫过成员头像和侧栏联系人列表（名字 + @角色 + 能力标签）
   字幕：Roundtable · IM 式多 Agent 协作平台

2. 旁白：我们把需求直接发进群里。注意，PM Agent 没有立刻动手——它先产出一份结构化计划。
   画面：New task → 输入"做一个产品定价页，含月付/年付切换"→ 发送 → 计划卡出现
   字幕：一句话需求 → 结构化计划（真实对象，不是一段话）

3. 旁白：每个任务有负责人、有依赖。没有我的批准，任何 Agent 都不会开工——人始终握着闸门。
   画面：点 Show plan 展开任务列表（assignee 头像、依赖标注）→ 点 Approve
   字幕：Human-in-the-loop · 计划经批准才分派

4. 旁白：批准后，工作流逐阶段推进。每个阶段一张卡：谁在干、干到哪、产出了什么，全部可见。右上角是执行平台——我们的适配器层接入了 Claude Code 和 Codex 两个平台，按角色绑定。
   画面：StageCards 依次翻面（Build 转 Review），鼠标停在平台 chip 上
   字幕：分阶段调度 · 多平台适配器层（Claude Code + Codex）

5. 旁白：产物是真实模型写的代码，一键复制；网页产物直接在聊天里渲染，可以交互。
   画面：展开 code 产物 → 复制按钮变绿 → preview iframe 里点月付/年付切换
   字幕：真实产物 · 聊天内即时预览

6. 旁白：更重要的是，这张桌子有记忆。我把 PM 的这条结论 pin 住，然后说一句没头没尾的话——"把切换做成能记住用户的选择"。
   画面：hover PM 回复 → pin 按钮高亮 → 输入追问发送
   字幕：手动 pin 关键消息 = 长期上下文

7. 旁白：它准确理解了我在说定价页的那个切换——因为聊天历史和 pin 的约束，每一轮都真实传给了模型。
   画面：新计划卡出现，放大 intake 摘要文字
   字幕：聊天历史 + pin 约束 → 自动注入模型上下文

8. 旁白：这套工作流本身是可定制的产品对象——阶段、席位、审批闸口都能改，改完直接驱动真实调度。新手用模板，重度用户搭自己的流水线。
   画面：Workflow 编辑器，拖动阶段、展示 Ship 阶段的 user-approval 闸口
   字幕：可定制 Workflow · 编辑即生效

9. 旁白：最后，每一次 Agent 之间的交接都是一张结构化卡片，并自动落盘。这个文件就是刚才那次运行真实写下的交接记录——带上下文预算审计，不是事后整理的。
   画面：打开 ai-logs/handoffs.jsonl，滚动并放大 context_audit 字段
   字幕：HandoffCard 自动落盘 · ai-logs/handoffs.jsonl

10. 旁白：分工、交接、依赖、质检，不再失传在聊天记录里。这就是 Roundtable。
    画面：回到首页圆桌全景，淡出
    字幕：Roundtable · 让 vibe coding 像一个团队

## 风格

- **语言**：中文旁白 + 中文字幕（界面本身是英文，字幕负责解释）
- **类型**：产品演示为主，第 4、7、9 段各嵌一句技术讲解（适配器层 / 上下文注入 / 运行时落盘），对着评分维度打
- **时长**：约 4 分钟（10 段 × 20–30 秒；dispatch 等待全部剪辑加速）
- **节奏要点**：两处必须特写放大——第 7 段的 intake 摘要（上下文记忆的证据）和第 9 段的 context_audit（AI 协作 30% 的证据）；这两幕是评委记住我们的地方
