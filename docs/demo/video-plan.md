# Demo 计划

## 目标

向评委展示 Roundtable 的完整真实链路——**一句话需求 → PM 结构化计划 → 人工批准 → 多 Agent 分阶段执行 → 真实产物预览 → 结构化交接记录**——并逐段对应比赛评分维度（AI 协作 30% / 功能完整 25% / 生成效果 20% / 创新产品感 10%）。核心论点：别的工具是"一个 Agent 干活"，Roundtable 是"一个 Agent 团队怎么协作不失控"。

## 页面 / App

- 本地地址：**http://localhost:3000**（`corepack pnpm ui:dev`）
- 录制前置（必须全过，详见 `docs/demo/storyboard.md` Pre-flight）：
  1. `docker ps` 确认 postgres/redis 在跑（否则 `pnpm dev:services`）
  2. `curl -m12 https://api.deepseek.com` 返回任意 HTTP 码即可（401 正常）
  3. 用 dev 凭据登录（登录态才有 workbench/pin/自建 Agent 全功能）
  4. 发一条丢弃用消息，确认计划卡**无 `degraded` 标记**
- 窗口 1280×800+，浅色主题，关闭系统通知，**全程不出现 `.env` / API key / Deploy 字样**

## 演示要求

1. 打开 http://localhost:3000，停在首页圆桌全景 2 秒
2. **自建 Agent**：侧栏成员区点 **Add an agent** → 表单里选角色、改名字、在 System Prompt 框敲一句、点亮两个能力开关 → **Add to workbench** → 新联系人行出现（名字 + @角色 + 能力标签，**放大联系人行** 1 秒）
3. 侧栏点 **New task**，在输入框先敲 `@` 让 **mention 菜单**弹出（停 1 秒），Esc 收起，再输入 prompt：`做一个产品定价页，含月付/年付切换`，回车发送
4. 等 PM 出计划卡（约 3–8 秒，**保留这个等待**，让评委看到"先计划不动手"）→ 点 **Show plan** 展开
5. 点 **Approve** → Build 阶段卡出现后**点 Stop 按钮** → 中断卡出现（Resume / Hand off / Discard 三个动作，停 2 秒）→ 点 **Resume** 继续 → 等 StageCards 走完（等待**剪辑加速到 5 秒内**，保留阶段卡翻面瞬间）
6. 鼠标指向阶段卡右上角 **平台 chip**（"Claude Code"），停顿 1 秒
7. 展开 code 产物 → 点右上 **复制按钮**（变绿色对勾，放大特写）→ 切到 preview 产物，iframe 里直接交互月付/年付切换（**放大**）→ 点 **Open in drawer** 全屏看一眼即关
8. **上下文一幕**：hover PM 回复点 **pin 按钮**（高亮特写）→ 在输入框发追问 `把切换做成能记住用户的选择`（一句没头没尾的话）→ 等新计划卡出现，**放大 intake 摘要**——它准确说出了"给定价页切换加状态持久化"
9. **IM 细节蒙太奇**（快剪，每镜 2 秒）：侧栏搜索框过滤任务 → hover 任务行点置顶（pin 图标浮顶）→ 点归档（折进 Archived 区）→ hover 消息点引用（`>` 注入输入框）→ 完成的 turn 上点 **Regenerate**
10. **登出态脚本剧场补两镜**【SCRIPT】：对话流里的 **HandoffCard 内联卡片**（结构化字段停 2 秒）→ **Diff 产物卡 + 依赖变更徽章**（上游产物更新，下游卡片亮 "dependency changed" 徽章，**放大徽章**）
11. 顶部进 **Workflow 编辑器**：拖一个阶段、点开 Ship 阶段的 user-approval 闸口设置，停 3 秒即可，不深玩
12. 切到编辑器/终端打开 **`ai-logs/handoffs.jsonl`**，滚动到今天的真实记录，**放大 `context_audit` 字段** 2 秒
13. 回到首页全景收尾
- 兜底：若 LLM 当场异常，步骤 3–8 全部改用登出态脚本剧场录制，台词去掉"真实模型"一句，其余不变
- 若必须压回 4 分钟：砍步骤 5 的中断恢复（留答辩现场演）和步骤 11 编辑器，其余保留

## 风格

- **语言**：中文旁白 + 中文字幕（界面本身是英文，字幕负责解释）
- **类型**：产品演示为主，第 5、8、12 段各嵌一句技术讲解（适配器层 / 上下文注入 / 运行时落盘），对着评分维度打
- **时长**：约 5 分半（13 段 × 20–30 秒；dispatch 等待全部剪辑加速；规则允许 3–10 分钟）
- **节奏要点**：三处必须特写放大——第 8 段 intake 摘要（上下文记忆的证据）、第 10 段依赖徽章（评委点名的差异化）、第 12 段 context_audit（AI 协作 30% 的证据）
- **诚实标记**：第 10 段两镜来自登出态脚本剧场（标【SCRIPT】），其余全部真实链路实录
